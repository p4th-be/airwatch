🌍 AirWatch Europe — Installation Django + MySQL

📋 Configuration Requise

  • Python 3.9+
  • MySQL 5.7+ ou MariaDB
  • pip (gestionnaire de paquets Python)

1️⃣ Installer les dépendances

pip install -r requirements.txt

Cela installe Django, MySQLclient, python-dotenv et requests.

2️⃣ Configurer MySQL

Créez une base de données MySQL :

mysql -u root -p
> CREATE DATABASE airwatch_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
> GRANT ALL PRIVILEGES ON airwatch_db.* TO 'airwatch_user'@'localhost' IDENTIFIED BY 'your_password';
> FLUSH PRIVILEGES;
> EXIT;

3️⃣ Configurer les variables d'environnement

Copiez .env.example vers .env et modifiez les valeurs :

cp .env.example .env

Puis éditez .env avec vos paramètres MySQL :

DB_NAME=airwatch_db
DB_USER=airwatch_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=3306

4️⃣ Créer les tables dans la base de données

python manage.py migrate

Cette commande crée les tables Django et l'app airquality.

5️⃣ Remplir la base de données

Option A : Données de base uniquement

python manage.py populate_data

Option B : Avec générateur de mesures (7 jours de données)

python manage.py populate_data --full

6️⃣ Créer un superutilisateur (optionnel)

python manage.py createsuperuser

Utile pour accéder à l'admin Django à /admin/

7️⃣ Collecter les fichiers statiques

python manage.py collectstatic --noinput

✅ Lancer le serveur de développement

python manage.py runserver

Puis accédez à : http://localhost:8000

📊 Commandes de gestion Django

  • populate_data - Remplir la base de données avec les pays, villes, stations
  • populate_data --full - Ajouter aussi des données de mesure générées
  • fetch_openaq_data - Récupérer les données temps réel depuis OpenAQ
  • fetch_openaq_data --city Paris - Récupérer pour une ville spécifique
  • createsuperuser - Créer un administrateur
  • runserver 0.0.0.0:8000 - Serveur accessible en réseau

⚠️ Points importants

  • Sécurité : Changez la clé secrète Django en production
  • DEBUG : Mettez DEBUG=False en production
  • Base de données : Sauvegardez régulièrement votre BD MySQL
  • API OpenAQ : Rate limited - pas plus de 10 requêtes/minute
  • Données : Les données générées sont aléatoires pour développement

✨ Prochaines étapes

  • Personnaliser CSS dans static/css/
  • Ajouter vos propres polluants/pays/villes
  • Connecter l'API OpenAQ pour données réelles
  • Créer des graphiques interactifs avec Chart.js
  • Déployer sur un serveur production (Heroku, PythonAnywhere, AWS, etc.)
